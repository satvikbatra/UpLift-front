# Contributing to UpLift Faculty Performance Management System

Thank you for your interest in contributing to the UpLift Faculty Performance Management System! We welcome contributions from the community and are grateful for your help in making this project better.

## Getting Started

### Prerequisites

Before you begin, ensure you have the following installed:
- Git
- Node.js (v18 or higher)
- npm or yarn

### Development Setup

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/your-username/uplift-faculty-performance.git
   cd uplift-faculty-performance
   ```
3. Install dependencies:
   ```bash
   npm install
   ```
4. Start the development server:
   ```bash
   npm run dev
   ```

## Code Style Guidelines

- Follow the existing code style in the project
- Use meaningful variable and function names
- Write clear, concise comments where necessary
- Ensure your code passes linting:
  ```bash
  npm run lint
  ```

## Pull Request Process

1. Create a new branch for your feature or bug fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. Make your changes and commit them with clear, descriptive messages following conventional commit style:
   ```bash
   git commit -m "feat: add new faculty evaluation feature"
   ```

3. Push your branch to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```

4. Open a Pull Request against the main repository

5. Ensure your PR description clearly describes the problem and solution

## Issue Reporting

When reporting issues, please include:
- A clear description of the problem
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Environment details (OS, browser, Node version)

## Code of Conduct

This project adheres to a code of conduct. By participating, you are expected to uphold this code:
- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback
- Respect different viewpoints and experiences

## Questions?

If you have any questions about contributing, feel free to open an issue or contact the maintainers.

Thank you for contributing to UpLift!
